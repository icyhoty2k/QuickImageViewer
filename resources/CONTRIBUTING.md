Contributing to QuickImageViewer
Thank you for your interest in improving QuickImageViewer. I prioritize clean, explicit, and performant code. To ensure the project remains high-quality and easy to maintain, please follow these guidelines when submitting features or fixes.

How to Contribute
Fork the repository and create your feature branch.

Code Style: Keep it simple and explicit. Avoid unnecessary abstractions.

Module Focus: Please keep your changes focused on the relevant modules:

WicDecoder.cpp: Use this for image decoding and format support.

Renderer.cpp: Use this for viewport and rendering logic.

FileHandler.cpp: Use this for file I/O operations.

License: Remember that all contributions are subject to the GNU Affero General Public License v3.0. By submitting, you agree that your changes will be released under this license to remain open and accessible to the community.

Submit a Pull Request: Please include a brief description of the feature or bug fix and how it improves the application.

I look forward to reviewing your contributions and evolving this tool together.
